주제 : 학생 관리 프로그램

(완료)	1. DB 사용자 계정 새로 생성 -> 계정명/비밀번호 자유
(완료)	2. 테이블명 : TB_STUDENT 
	컬럼 1) STD_NO         NUMBER (PK)                     학생 번호
	       2) STD_NAME     VARCHAR2(20) NOT NULL     학생 이름
	       3) STD_AGE        NUMBER  NOT NULL           학생 나이
	       4) STD_GENDER  CHAR(1)   CHECK                학생 성별('M' , 'F' 제한)
	       5) STD_SCORE    CHAR(1)   CHECK                 학생 성적('A','B','C','D','F' 제한)
	(컬럼추가) STD_COMMENT VARCHAR2(4000)	      학생 특이사항

	3. 화면 구성
(완료)	1) 메인페이지 
  (완료)		- 등록되어있는 모든 학생 목록 출력(이름, 나이, 성별, 성적)
  (완료)		- 학생 목록에서 이름 클릭 시 학생 상세조회 화면 이동
  (완료)		- 학생 추가 버튼 (클릭 시 학생 추가 화면 이동)
  (추가사항1)	- 학생 전체 수 추가 (남성 여성 인원도 별도 출력)
  (추가사항2)	- 학생의 성적분포 추가 (A, B, C, D, F)

(완료)	2) 상세조회 화면
  (완료)		- 선택된 학생에 대한 상세 정보 출력(고유번호/이름/나이/성별/성적)
  (완료)		- 목록으로 버튼
  (완료)		- 수정 버튼 (클릭 시 수정 화면으로 이동)
  (완료)		- 삭제 버튼 (성공 시 메인페이지로, 실패 시 상세조회 화면으로)
  (추가사항1)	- 상세조회 시 특이사항이 있는 경우 출력
  (추가사항2)	- 성별이 "M or F" 가 아닌 "남성(M) or 여성(F)" 으로 보이도록 변경

(완료)	3) 학생 추가 화면
  (완료)		- 이름/나이/성별/성적 입력 할 수 있는 태그 작성(여러가지 사용해보기 ex.radio, checkbox, select 등)
  (완료)		- 추가 성공 시 메인페이지로, 실패 시 추가 화면으로
  (추가사항1)	- 추가 시 특이사항을 적을 수 있게 했음
  (추가사항2)	- (JS) 추가 시 이름이나 나이 잘못입력하는 경우 제출막기
  (추가사항3)	- 목록(메인페이지)으로 돌아가는 버튼 추가

(완료)	4) 수정 화면 
  (완료)		- 이름/나이/성적 수정 가능하도록 해야함
  (완료)		- 기존 정보 출력되어 있는 상태여야 함
  (완료)		- 수정 성공 시 메인페이지로, 실패 시 수정 화면으로
  (추가사항1)	- 수정 시 특이사항을 고칠 수 있게 했음(특이사항 작성되어있든 안되있든 화면에 나타남)
  (추가사항2)	- (JS) 수정 시 이름이나 나이 잘못입력하는 경우 제출막기
